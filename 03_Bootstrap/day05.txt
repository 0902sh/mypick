Day5

1. Webflow
(1) low-coding, no-coding
    coding을 최소화하면서 결과물을 만들어내고자 하는 용어
    coding을 적게 하더라도 html, css의 기초 없이는 제대로 활용 어려움

(2) no-coding website 디자인의 대표적인 사이트로 Webflow가 있음

(3) 코딩의 진화
   카드(천공) -> 편집기 -> 키워드 강조 편집기
   -> 통합개발툴(IDE) -> 웹 기반 코딩 -> 노 코딩

(4) Webflow는 드래그&드랍 방식 + 설정 변경
    으로 웹 디자인을 할 수 있는 툴
    : 한글 사용이 아직 불편함
      -> 우리나라에서는 아직 많이 알려지지 않은 상태
      -> 향후에는 대세가 될 것으로 예측

    => 가장 큰 장점은 lecture가 풍부하게 예측됨

2. 반응형 레이아웃(Responsive Layout)
    - 웹의 레이아웃을 만들 때 사용자의 해상도(가로사이즈)를 고려
    - 스마트폰이나 태블릿 등은 화면이 작기 때문에 가독성이 중요
    - 이런 문제를 해결하고자 하는 방법 중 하나로
      해상도에 따라 배치를 변경하는 레이아웃
    - @media를 통해 구현

3. 부트스트랩(Bootstrap)
    - 세계에서 가장 인기있는 모바일 우선형/반응형 프론트엔드 프레임워크
    - 프레임워크
        패키지 : 소스 + 디렉토리
        라이브러리 : 여러 개의 패키지
        프레임워크가 제공하는 공간에 라이브러리 탑재 가능
    - 가장 빈번하게 사용되는 CSS, JS를 미리 만들어놓고
      CDN 방식이나 다운로드 방식으로 사용하도록 서비스

(1) 설치 방법
    - getbootstrap.com 사이트에서 다운로드 > 압축 해제
    - CSS 방식으로 링크시켜서 사용

(2) 탄생 배경
    - 트위터 프로젝트에서 시작
    - 서로 다른 인터페이스를 사용한 여러 개발자의 공동 작업
    - 디자인 불일치, 관리 난이도 상승, 코드 방대
      -> 일관성 있게 관리 불가능
    - 트위터 개발자와 UI 디자이너가 최초로 개발

(3) 부트스트랩
    - Front-End 개발을 빠르고 쉽게 할 수 있는 프레임워크
    - HTML, CSS 기반의 템플릿 양식, 버튼, 내비게이션 및 기타 요소
    - 트위터에서 사용하는 각종 레이아웃, 버튼, 입력창 등 디자인을
      CSS, JS로 구성
    - 프레임워크이므로 jQuery
    - 상업적 이용 가능
    - 모바일/PC/태블릿 디자인 지원
    - 검색엔진을 통해 bootstrap template로 검색하면
      상당한 양의 데이터가 검색됨
